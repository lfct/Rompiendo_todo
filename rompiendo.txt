REPASO TOTAL CURSO GIT Y GITHUB

Esta es mi primera practica con git después 
de terminar todo el curso de git y github
entonces iniciare con los primeros comando
como los son git init, status, add, rm, rm --cached 
, commit, config  --global user.name "nombre",
config --global user.email "correo", cat.

para la eliminación de archivos en el area staging
y del proximo commit usamos el comando git rm --cached rompiendo.txt
pero los mantiene en el dico duro, para liminarlos de 
git y del disco duro, para liminar del disco duro se
usa el comando git rm --force, finalmente git reset
HEAD para sacar archivos del areas de staging, no se borran.

GITHUB:

Github es un servidor remoto de repositorio "red social de programadores", trabajos 
localemente y enviamos o traemos el repositorio remoto
para ello existen varios comando importantes.

git clone url_del_servidor_remoto: nos permite descargar los archivos 
de la última versión de la rama principal y toda la historia 
de commit en la carpeta .git 

git pull: traemos todo del repositorio del servidor remoto a nuestro
repositorio local.

git push: enviamos del repositorio local al respositorio remoto.

clone or download (HTTPS): Trae todas las cosas de github a git.

git remote add origin https://git... : decirle a git que vamos
agregar un origin para hacerle pull (traer) y push (enviar).

git push origin master: git enviale a origin la rama master.

git pull origin master : traemos de github (origin) a git (master).

git pull origin master --llow-unrelated-historias: fusiona las histo-
rias de github (origin) con con git (master).
